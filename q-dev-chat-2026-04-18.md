ApiService.updateBankAccount(account.accountId, {

- `getOldestTransactionDate`: 本日（JST）= `2026-04-18`

**① `BankAccountManagement.vue` の `updateSortOrder`**
`sortOrder` のみ送信するよう変更。

`new Date()` をJST基準に統一します。

```javascript
// 変更前
return new Date().toISOString().split('T')[0]  // UTC基準

// 変更後（JST = UTC+9）
const now = new Date()
now.setHours(now.getHours() + 9)
return now.toISOString().split('T')[0]  // JST基準
```

また `recalculateFromDate` 内で `startDate > endDate` の場合は `endDate = startDate` にフォールバックする安全策も追加します。

## Promise.all とは

`Promise.all` は、複数の非同期処理を**同時に並列で実行**する仕組みです。

```javascript
// Promise.all の場合（全件同時に送信）
await Promise.all([
  PUT /bank-accounts/口座1  ┐
  PUT /bank-accounts/口座2  │ 18件が同時に飛ぶ
  PUT /bank-accounts/口座3  │
  ...                       ┘
])
```

```javascript
// for ループの場合（1件ずつ順番に送信）
for (let i = 0; i < 18; i++) {
  await PUT /bank-accounts/口座i  // 1件完了してから次へ
}
```

---

## Lambda の同時実行数上限とは

AWS Lambda には**同時に実行できる関数の数に上限**があります。

- デフォルトの上限: **リージョン全体で1,000件**
- ただし今回の問題は別の理由です

今回の本当の原因は、**Lambda のコールドスタート**です。

18件を同時に送信すると、Lambda が18個のインスタンスを一気に起動しようとします。起動が間に合わないインスタンスはリクエストを処理できず500エラーを返します。

```
18件同時リクエスト
  → Lambda が18個同時起動しようとする
  → 起動が間に合わないものが発生
  → 500エラー（CORSヘッダーなし）
```

---

## なぜ for ループで解決するか

1件ずつ順番に送ることで、Lambda が1件処理してから次のリクエストを受け付けるため、起動の競合が発生しません。

---

## 補足・正直なところ

厳密には Lambda の同時実行数上限（1,000件）には全く届いていません。今回は**コールドスタートによる起動遅延**が原因でした。「同時実行数の上限」という説明は正確ではありませんでした。申し訳ありません。
